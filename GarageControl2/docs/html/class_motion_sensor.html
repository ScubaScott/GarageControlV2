Detailed Description

PIR motion sensor with debouncing and relay spike rejection.

This class provides:

  • Digital debouncing to filter electrical noise and prevent fluttering
  • Motion detection via HIGH signal from PIR sensor module
  • Forced acknowledgment to suppress false triggers from EMI/relay spikes
  • Timeout-based activity tracking for configurable-duration occupancy detection

The forced acknowledgment mechanism is called by the GarageLight controller immediately after relay activation, preventing the electromagnetic pulse from being misinterpreted as human motion detection.

Note
Debouncing is simple HIGH-to-LOW state verification. For more sophisticated filtering, examine the implementation for duration-based requirements.

void MotionSensor::forceAck ( )

Forces acknowledgment to suppress false motion triggers.

Forces acknowledgment to suppress false triggers from relay spikes.

Call immediately after relay activation to prevent the electromagnetic pulse from being detected as motion on the next poll() cycle.

Motion detection is suppressed for a short duration (see implementation). This is called by GarageLight::turnOn() and must be called before electromagnetic relay actuation to be effective.

Note
The forced acknowledgment window duration is hardware-dependent. Pin configuration and debounce timing affects effectiveness.

bool MotionSensor::isActive ( )

Gets the current real-time motion sensor pin state.

Gets the current active state of the sensor.

Returns the instantaneous GPIO pin reading without debouncing. This is NOT the same as motion detection; it's the raw sensor output.

Returns
True if PIR sensor is currently reading HIGH (motion detected now), false if reading LOW (no motion at this instant)
Note
This should be used sparingly. For time-windowed queries, use recentlyActive() instead.
Returns
True if sensor pin is HIGH, false otherwise.

bool MotionSensor::poll ( )

Polls the motion sensor for new motion activity.

Polls the motion sensor for activity with debouncing.

Should be called once per main loop iteration (typically 100-200ms). Returns true only once per motion event, even if PIR signal remains high. Motion detection is suppressed during forced acknowledgment window.

Returns
True if NEW motion detected and not in forced-ack window, false otherwise (including during suppression period)
Note
This method implements debouncing by verifying the pin state has stabilized HIGH. See poll() implementation for timing details.
Returns
True if new motion detected and acknowledged, false otherwise.
